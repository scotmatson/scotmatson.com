Setting Up a Development Environment

Installing the tools

Chapter overview. This section will require a programming text editor, a host server, and a mobile emulator.

Setting up a local web server

Setting up a local web server using Microsoft Internet Information Services(IIS) server. The reason this is necessary is due the fact that the mobile phone emulators cannot open up local files from the system. They are virtual machines that run as if they were on actual phones. Using a local web server is a work-around to this problem. Following along with the video, I installed IIS7, created a MobilWebDevFund folder, and tested it out on my local host to make sure it was working properly which it was.

Installing device emulators

Installation of the Windows, Android, Opera, and iPhone mobile emulators. Each is tested through the local host within the native emulator web browser. Each environment appears to be a large download and not very intuitive. You would think a single application would exist for developers to test their code on.

Using device emulators

Emulator walkthrough. Covers softkey functionality along with a number of useful keyboard shortcuts and also how toggle the keyboard and enter landscape mode.

Downloading Modernizr and Mobile Boilerplate

Discussion on Modernizer (a JavaScript library that lets you test technologies within a browser before you use it) and the difference between the development version and production version of the software. Finishes up with HTML5 Boilerplate, a simple template which is a collection of best practices for modern mobile browsers.

Building a first mobile web page

Takes a basic html file which includes an h1 tag and a p tag and shows the default presentation on mobile browsers. Default appears zoomed out and difficult to read without zooming in. By placing three meta tags in the head of the html document you can tell the browser to load the html content to fit to screen on load. These meta tags are HandhelfFriendly, MobileOptimized, and Viewport.

Adapting Desktop Sites for Mobile Devices

Reviewing mobile markup languages

The evolution of mobile markup langauges. SGML developed in the early 1970's gave rise to HTML, XML, and HDML. HTML and XML came together to produce XHTML in the mid-90's. In the late 90's cHTML, XHTML basic and WML emerged. In the early 2000's all of these languages converged to become XHTML-MP and finally HTML5 which is now the standard of today in desktop browser and mobile browser development. Understanding the history of these languages helps developers understand what they may encounter during their career.

Understanding content adaptation approaches

Three ways of adapting your web sites for a mobile and desktop environment. The Do Nothing approach does as the name said, no changes are made to adapt the layout for mobile production. The multi-Serve approach uses one document with complex scripts and sniffing methods to target various form factors. Finally a Mobile-Specific approach uses two separately developed sites, one for desktop and one for mobile and then uses sniffing to discover how you are accessing the web site and serves you up the appropriate platform specific site.

Controlling the viewport layout

Review of meta tags used to make desktop pages mobile ready. HandheldFriendly is used to tell the browser that the site is already optimized for mobile devices and the browser does not have to do any extra work. MobileOptimized is more advanced than Handheldfriendly and is used to specify the width of screen the site is optimized for. Viewport was designed by Apple and has become the standard for controlling how the mobile browser displays a page. When designing web sites all three meta tags should be used as modern browsers are intelligent enough to know which one to use.

Designing forms

To optimize forms for mobile use, minimize the number of screens a user has to pass through. Use native web form controls instead of custom ones. Use top-aligned form labels instead of left or right input fields. Give the option to display password characters. Use HTML5 form input types.

Using CSS media queries

CSS media queries provide a way to selectively apply a style sheet based upon certain criteria of the media where the content is to be displayed. These criteria range from window size, screen size, aspect ratio, color deptch, and resolution. Many of these accept min- and max-, also, all css compliant units of measurement. To apply the media query you can specify a media attribute in a link element to an external stylesheet and state screen AND (max/min width). This can also be declared within a stylesheet by using the @media directive which states the same thing, screen and (min/max width......).

Getting Started

Learning HTML

HTML is surprisingly easy to learn. The basics can be picked up as quickly as five minutes. HTML is a mark-up language, this means that tags<code> are used to describe the content on the page.

Modern web design uses different languages together to create feature rich web site. HTML: structure, CSS: presentation, JavaScript: behavior, PHP: back-end, CMS: content.

Choosing a code editor

Authoring HTML requires little more than a simple text editor. Special editors exist help developers speed up the process by integrating syntax highlighting, line numbers, and special language support and formatting

It is recommended that sever code editors are downloaded and experimented with to find one that fits your personal preferences. James gives a walk-through to the Komodo IDE at this point.

Exploring basic HTML syntax

HTML uses tags to identify page content. A tag uses a left angle bracket < and a right angle bracket > to hold information about the structure of the web page. This is an element of HTML. <p> for instance tells the browser that we are creating a paragraph while </p> tells the browser that we are done using the paragraph tag. The first p tag is called an opening tag while the second p tag is known as the closing tag.

Tags can also have attributes which tell us more information about a particular tag being used. An example would be <p lang="en">. The lang="en" attribute tells us that this paragraph will be typed in English.

Tags can be placed within other tags creating a parent/child relationship. The key thing to remember is when closing tags, last in is always first out.

When creating a web page a few tags must be present. These are the DOCTYPE declaration, the HTML tag, the head tag, and the body tag.

Do I need to learn HTML5

1991, Tim Berners-Lee introduced the first HTML tags. Over the next decade multiple versions of HTML were released and often had unique syntax across multiple browsers. The following decade was spent attempting to standardize HTML to ease the frustrations of programmers and developers. Currently HTML5 is being implemented and developed. HTML5 is not necessary that much different from its predecessors but it has one key difference in that its heavily focused on integrating applications and working closely with CSS and JavaScript.

Exploring HTML references

HTML resources are introduced to help find information and understand standard practices, syntax, and organization. The two most well known groups that are actively involved in development is W3C and WHATWG. A site known for providing great developer tutorials and assistance is the Mozilla Developers Group.

Basic Page Design

Exploring an HTML document

One of the best ways to learn about HTML is to look at other developer's code. This can be easily accomplished by opening a web browser and opening the 'inspect element' or 'source code'. For new users this still may be difficult to understand, but most browsers have built in features which allow you to use your cursor to highlight various elements of the page which will in turn highlight the source code which is being viewed, and visa versa.

HTML documents are made up of three major pieces. 1: The HTML tag is the parent tag of the entire document. Without it, you don't have an HTML document. 2: The document head contains meta data, links, and styling. It is not necessarily information presented on the page itself but it is read by the computer. 3: The body element holds information that is to be displayed on the web page itself.

Working with DOCTYPE declarations

The <!DOCTYPE> triggers a set of standards to ensure the page is rendering properly. Multiple versions exist and pre-HTML5 can be difficult to remember. Existing web sites have DOCTYPES which can copied and pasted into an HTML file.

The <!DOCTYPE HTML> is what is used for HTML5. The DOCTYPE will be the first thing entered into code on an HTML file and is found at the very top of the HTML file. It is not something that is visible to the user through a web browser but can be seen by viewing the source code.

Examining the document head

After the opening HTML tag lies the document <head>. Between the opening and closing head tag lies invisible information that details information about how the document will work. Information such as <meta> can provide useful information to write a description to a page or use keywords, to write how a page should be viewed through multiple platforms, or instructions on which character set to use.

The meta tag is helpful for search engine optimization, users, and browsers. When used it's worth noting that meta tags do not require a closing meta tag. Aside from the meta tags a opening and closing <title> tag which displays information in the browser title bar can be found, styles for formatting, and scripts as well.

Looking at the document body

The <body> tag holds all of the information that is to be presented to a user through the browser. Anything typed into the body will be displayed but it is important to remember that unless tags are applied, no structure or formatting may be added.

Typical elements of the body that may be found are very similar to that of a word document. Those structural elements are a header, main content, side content, and a footer.

Adding document structure

Given the freedom is design, you could teach HTML structure to multiple designers and give them content and you will end of up with many different looking web sites. One of the most important things to consider though is does the layout meet HTML standards and will it be easy to upkeep.

When building sites, aside from building a structure that is easy to maintain, you want to have consistency through the entire site. This means that as you browse from page to page you see common themes which help users know that they are still on the same page and the page will have consistent functionality.

Content can be more easily managed by dividing related sections with an opening and closing <div> tag. Since having a page full of div tags does not allow much room to individually style different sections we often will apply an id="" attribute to isolate isolated sections or a class="" attribute to isolate groups of related content. By using these attributes it becomes much easier to apply CSS stylings and HTML structure.

With the induction of HTML use of the div tag has been drastically reduced as we now have special tags like <header>, <section>, <article>, <aside>, and <footer>.

Formatting Page Content

How does HTML format text?

As a mark-up language, HTML uses tags to format the content on a page. Without tags a browser will render text to the browsers default setting. With the inclusion of tags we will not necessarily see a visible change to the format of the content in the web browser, as that is not HTML's purpose. What we will see is a change of how content is structured.

Content heading tags are used to tell the browser that this line of text is a heading, and also can indicate the importance of the content. Six heading tags are used with 1 being the most important and 6 being the least important. They are used as an opening and closing tag and typed <h1>. A <pre> opening and closing tag will present text exactly how it is layed out in the HTML file. The <em> and <i> opening and closing tags both format the text into italics but one is used for visual effect while the other is used to emphasize text and will be semantically understood by a screen reader.

Adding headings

Headings are used to structure content and develop a hierarchy of importance, <h1> being the most important and <h6> being the least important. Browsers will denote this importance by the size of the size associated with the heading but this is only a reference. Proper sizing should always occur through CSS only.

Planning ahead helps develop a proper heading strategy. Typically only one h1 tag will exist and indicates the title of the page. As content is presented it is a best practice to use headings sequentially, so no jumping from an h2 to an h5 tag. Proper is of heading tags can help SEO better catalog the information on your page.

Formatting paragraphs

Most browsers will default any free floating text into a paragraph tag. The <p> tag is probably one of the most used tags in HTML. When the paragraph tag is placed around a block of text it tells the browser that this text is a paragraph and belongs together and also formats the text to have a distinguishable amount of white space above and below it.

Controlling line breaks

A <br> is a line break tag and forces a hard return to occur anywhere it is placed. It is useful to place these in places where you would like to place groups of code on separate lines without losing the relationship that the code shares with itself, such as an address wrapped within a <p> tag.

Creating lists

Depending on how content needs to be organized, three different types of lists exist in HTML. 1: An ordered list <ol> uses an opening and closing tag and is used for list items that would not make sense when placed in a random order. The list created will be numerically ordered. 2: An unordered list <ul> is used for a list that still makes sense when list items are placed in a randomized order. It uses bullet markets to indicate that each item of the list is in fact a list, rather than a number. Ordered and unordered list items are creating by placing opening and closing list item <li> tags with list content within the ordered or unordered list tags.

Finally the third type of list is called a definition list <dl>. These are lists that use terms <dt> and definitions <dd> instead of list items. While it presents similarly to lists, it is more specialized to presented paired information that explains or gives additional detail such as a glossary, dictionary, or FAQ.

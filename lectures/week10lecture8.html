Programming Basics

What is programming?

"A computer program is a set of instructions..." while the art of proramming is taking a larger idea and breaking it apart into individual steps. Programming is as simple as a step by step series of directions which linearly guide a computer from point A to point B and using a programming language to write code to communicate with the computer.

Programming languages use statements to form instructions for the computer to interpret. Statements in a computer language are similar to sentences in the English language. While statements vary from language to language, the statements all tend to function the same by using a few symbols or words to state a simple task.

What is a programming language?

Machine code is the foundation of programming languages. It varies from CPU to CPU and consists of millions of lines of code depicted by a series of letters and numbers that is intended to be read by computers rather than humans. Programming languages used my people help bridge the gap between the machine code computers use and the languages used by humans, also known has source code.

Since programming languages have been used, hundreds have been developed, but with each innovation only a handful of languages remain popular at any given time. Programming languages can be classified as high level programming languages and low level programming languages. A high level programming language tends to be easier to learn and apply as it uses statements which closely resemble modern day languages, they also are easier to use across multiple platforms. The downside to high level programming languages is that they often create slower applications due to the fact that they require a greater amount of processing power to convert them into machine code. In contrast, a lower level programming language uses more abstract syntax and tends to operate more efficiently. Assembly language is the lowest level of programming language used by people, a step above machine code.

To start writing any programming statement, three things must be understood. First, how to start writing the source code, secondly, where the source code will be translated into machine code, and third, how do we actually execute the program.

Writing source code

Programming languages are written using nothing more than a plain text file with an extension which correlates with the programming language used such as .html, .ruby, or .java. When compared, many statements in programming languages are very similar if not exacty the same. This is often due to a shared history in their development.

Multiple applications exist to assist programmings in the development of source code. Programmer's text editors add such features as line numbering, highlighted syntax, auto-completion, syntax checking. Integrated development environments are larger programs which act as text editors but offer many more features. They are generally used when you begin concetrating your code to one particular type of area.

Compiled and interpreted languages

There are two ways to transform source code into machine code, compiling, or intepreting. A compiled language uses a program called a compiler which goes through the source code and saves a separate file which is written in machine code which is to be used by the computer. Interpreter's on the other hand work on the fly. They process code line by line and do not save it as a separate machine code file.

Compiled code can be easily given to other users and runs more efficiently. Although it can only be used by the platform is was written for. Interpretive languages are slower, and the source code tends to be public, but they are able to be used on multiple platforms.

Although the most common way to convert a language to machine code is either through the afore mentioned methods, a third way of converting source code into machine code does exist. Byte code combines the best qualities of compiled languages and interpretive languages. This is accomplished by partially compiling a language into an intermediate language which is later converted machine language by an interpreter when the application is executed. This is sometimes refered to Just-In-Time compilation or "JIT code".

Core Programming Syntax

Why JavaScript

JavaScript was chosen as the core language for this course do to its accessibility by users with little to no experience with programming. It also requires no additional software to be downloaded. JavaScript is a language that works with and manipulates websites, it was written specifically for that reason. JavaScript has no affiliation with Java and is sometimes referred as a scripting language. A scripting language is a programming language that is embedded inside another application, in this case, a web browser.

Scripting language tend to be more limited than general purpose programming languages but are also more easily learned. JavaScript is an interpreted language which is translated by the web browsers JavaScript rendering engine as the website loads. When writing JavaScript code it is important to remember that it is a case sensitive language and close attention to detail is a must. This case sensitivity along with other subtle syntax clues (such as curly braces and semi-colons) show a relationship with the C programming language from which JavaScript was derived.

Variables and Data Types

Introduction to variables and data types

When we write programs we are creating types of data that we need to track. This data is stored in a variable which is a named entity which acts like a container for a data value. In JavaScript a variable is declared by using "var" followed by the variable name that we choose. These variable names must be written as one word with no spaces included, they may consist of letters, numbers, dollar signs, or underscores, although they should begin with a letter.

Variables can be created without assigning it any properties, this causes the computer to dedicate a piece of its memory to this variable, assuming that something will be stored there later on. Assigning data to a variable is as simple as typing the name of the variable and following it with an equals sign a.k.a. the assignment operator, and then typing a character, String, or number. This operation can occur on a single line or multiple lines. When a String is assigned to a variable, remember that JavaScript is a case-sensitive language.

Understanding strong, weak, and duck-types languages

Most programming languages require not only the declaration of a variable but the data type as well. Data types include Strings, Chars, Doubles, Ints, Booleans, Floats, etc... A language with these requirements is referred to as a strongly typed language. JavaScript is a weakly typed language. This means when a variable is declared, we do not need to declare the data type, it will look at our code and figure out what we mean on its own. While their are a number of different data types, JavaScript uses only numbers, Strings, and Boolean values. While a weakly typed language seems like it would be easier to use, the major disadvantage is that we don't always know right away how the computer is referring to a specific data type; this may ultimately lead to errors in our programs.

Using characters and strings

Anytime we contain data in a set a quotations we are creating a String. Double quotes or single quotes are okay to use but cannot be mixed with one another. We may contain a value within double quotes and use single quotes within the actual data itself or visa versa. If we require the use of double quotes with a set of double quotes we must use an escape character to tell the computer not to prematurely end the String. This is performed by placing a backwards slash up against the value we wish to nullify.

Various functions exist which let us find out information about our data. With Strings for instance, by typing the name of the variable followed by the method length such as name.length, this will return the number of characters within the String.

Working with operators

Different types of operators are used to assign values to variables or make comparisons or calculations between values. A single equals sign is the assignment operator and places the data to the right of the equals sign and assigns its value to the variable located on the left of the equals sign. When performing calculations we may use + for addition, - for subtraction, * for multiplication, and / for division. Programming languages follow the same conventions as the mathematical order of operations by performing * and / calculations prior to + and - calculations. If we wish to override this precedence parenthesis may be used to indicate "do this first".

Since working with numbers is a fairly common task when writing code, a number of shorthand notations exist to make these tasks more easily performed. By placing a mathematical operative sign followed by the assignment operator such as +=, we are stating that we want the value to the right to be added to the value on the left. This notation is also true of -, *, and /. Lastly, the increment and decrement operator are used to increase or decrease an integer by "1". This is commonly used when writing loops. To write this code you first use the name of your variable, followed by a ++ to increment, or -- to decrement.

Properly using white space

JavaScript, similar to all programming languages, ignores whitespace. It has no impact on the behavior of the code whatsoever. We can write code as compact or expanded as much as we'd like but we must ensure that spaces do occur between different elements of a statement so the computer, such that var b=2 does not equal varb=2 as the computer would read this as varb rather than var b. The only two places that whitespace matters is in the elements of code themselves and also within Stirngs, otherwise whitespace is used for nothing more than readability. Adding comments to code for human understanding

In JavaScript commenting out lines of code or adding notation is performed by using // for single lines or /*for block of code*/. Commenting is useful for debugging code, temporarily removing chunks of code when testing, or just to add version control and explanation to help us remember what various and programs do.

Understanding the Mobile Web

Understanding the mobile context

Mobile web development is not just about the platform itself but also how and when the technologies are implemented. This requires developers to take new variables into consideration such as location of use, environmental factors, and activities occurring doing use. When form factors are being considered however it is important to note the differences in screen size and what input devices are being implemented by the user.

When designing for a mobile enviroment you can ask yourself a number of questions to ease the process.

  • Who is using the site
  • What are users doing when they access the site
  • Where is the site being acccessed from
  • How is the user accessing the site
  • When is the site being accessed
  • Why is the user accessing the web site

A survey of mobile sites

  • Twitter: Well defined links, consistent use of fonts, minimal use of graphics, simple and clean design.
  • Yahoo: Optimized navigation, consistent use of fonts, well defined links and search utilities.
  • eBay: Minimal use of graphics, clean layout, consistent use of fonts, clearly defined navigational links and search utilities, optimized for vertical scrolling

  • Wikipedia: Minimal use of graphics, limited use of color, clearly defined navigational features, optimized for vertical scrolling, optimized for reading and navigation
  • Ars Technica: Clearly defined icon navigation, clearly defined header and summary article presentation, optimized vertical scrolling

Targeting mobile browsers

Understanding the rendering engines and the platforms which use them. Current technologies offer various levels of support across the available platforms. Different approaches must be used to accomodate the most widely used browser application and making web sites available for antiquated versions as well. Rather than targetting individual browsers the author recommends grouping the browsers into various classes based upon their capabilities and designing sites to meet their needs.

Mobile Web Development Guidelines

Designing for one web

By following a simple set of guidelines and with some careful planning, designing web sites which function properly with older technology, current technology, and future technology should be easy. These guidelines should be utilized to help accomplish this task: keeping content separate from style sheets and script files, avoid platform specific mark-up, avoid in-line styles and in-line JavaScript statements, pick a baseline of functionality, implement progressive enhancement.

Using well-defined web standards

Adhering to baseline web standards and proven specifications such as XHTML 1.0, HTML5, and CSS3 help optimize the accessibility and functionality of your web site. For more advanced technology, provide fallback support for browsers with limited capibilities. As much as possible, rely on client-side scripting over server-side scripting and implement proven JavaScript libraries such as Modernizer or jQuery to help improve the usability or your site. Server-side scripting may be used for detecting the mobile platform in use but it is considered bad practice to implement browser sniffing. Furthermore, many new features now exist such as media optimization, local storage, and geocaching which enhance the user experience.

Designing mobile-friendly pages

When designing mobile sites, consider how the user will interact with the site. It is best to design for the finger rather than a styus as the practices used for stylus design often is not compatible with the functionality provided by the user a touch interface. Typically 40-80 pixels need be alotted for clickable targets. Pages should be optimized for vertical scrolling as most mobile devices are utilized with a vertical dominant interface. Certain technologies should be avoided completely on mobile devices, these include framesets, tables, image maps, and plug-in dependency.

Being crisp, clean, and succinct

Certain best practices should be followed to simplify site designs and focus on content. Using fewer colors and font faces will enhance readability. Avoiding excessive graphics and unnecessary interactivity which tend not add any level of interactivity, this also includes background images which decrease readability and rendering capabilities. Images should be optimized to accomodate various resolutions and judicious use of white space to enhance readability and navigation is encouraged. Providing a link to the desktop version of a web site gives users access to services which may not have been possible to implement for a mobile site.

Minimizing input where possible

Due to the interface used in mobile browsing, it is important to properly plan site design to minimize any potential disruptions in the user experience. One point of focus is sites which require user data such as financial institutes, travel agencies, or eCommerce. The more fields a user must manually fill out the less likely it will be for them to continue to use that platform interface. By making use of drop down menues, default values, and DOM storage, you can limit the amount of input required by the user and increase the usability of your site. Utilizing mobile capabilities such as geolocation services and datetime can greatly minimize user input and enhance the overall experience.

Focusing on the core scenarios

When planning to design a mobile site, first consider just a few things that clearly define what your site is used for, this will guide the development process. Next, attempt to focus on the perspecive of how a new user will navigate the site, you want to ensure that the services most expected to be utilized in this context are clearly defined and accessible by all users. Realizing the limitations of the mobile platform and not attempting to important every feature of the desktop platform will give a more natural flow to the mobile site and help aid in a more consistent and enjoyable user experience.

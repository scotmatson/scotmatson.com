Putting It All into Practice

Taking a look at the finished site

Walking through a desktop site helps better understand what can and cannot be transfered to the mobile environment. Large images, drop-down navigational links, in-line forms are just a few examples. Once a thorough evaluation has been peformed, take a look at the HTML document. It is important that the HTML document is written in pure HTML, free of JavaScript, and embedded objects... these levels of functionality should be linked through separate documents of their own. This will allow a clean transition to other environments by creating new CSS/JS documents that will be called upon when specific device/screen criteria are met.

Examining the header and background image style of the Home page

Comparison of the desktop and mobile CSS. Also covers jQuery JavaScript code which shows that the JavaScript will not function unless the screen width meets a specified dimension. By linking multiple CSS stylesheets and specifying rules for the activation of JavaScript it becomes easy to reuse our HTML document without requiring any restructuring. The process should retain site functionality while being transparant to the user.

Examining the hover effect and layout styles of the Tours page

More CSS comparisons. Interesting point was mobile browsers support the hover class through holding a finger upon an element. This feature was implemented to support older sites since the touch screen interface is still a newer technology.

Examining mobile forms on the Contact page

CSS review of form elements on desktop site and mobile site. Creating a single column layout with a narrow width on the mobile site made the form layout easy to view and interact with.

Viewing and testing the mobile site on emulators

Testing mobile site on design on desktop mobile emulators. HTML5 form elements add usability while it appears the mobile CSS have maintained the sites function and access.

Viewing the site on devices

Final test phase, opening the site in actual mobile devices.

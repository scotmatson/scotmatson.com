Creating Mobile-Optimized Pages

Detecting client capabilities with script

When detecting client capabilities, avoid using 'browser sniffing' and instead use JavaScript to detect client features which are present. Using a prewrittten JavaScript library designed towards feature detection like Modernizr can cut down a significant time on site development. Modernizr can be set up to evaluate which JavaScript, HTML, and CSS features you support and either implement those features or instead turn them off so a fall-back option may be used instead.

Caching information with local storage

With HTML5 a new type of client-side local storage was implemented. Previously this task was handled by 'cookies' which were often limited in size, insecure, and inefficient. HTML local storage functions similarly to a database and can be easily programmed with familiar get/set functions commonly used by popular programming languages. Since not every user will support the local storage API, modernizr once again can be used to detect if this feature is present, if not... fall back to browser cookies.

Determining position with geolocation

The geolocation API is a feature rich application which is commonly used for mapping or filling user defined requests such as what local business are in the area and sharing personal information through social networking applicatoins. While implementing this API the mobile device has the ability to offer up user information such as latitude, longitude, zip code, elevation, etc... Once again detection through modernizr makes this API much easier to implement. Geolocation still can require a great deal of coding depending on how it will be used. Microsoft, Google, and mapQuest all offer their own flavor of geolocation API with walkthrough guides.

Minimizing HTTP requests with data URLs

A data URI is represents binary data in a page as a String. It is commonly used to reduce the number of HTTP requests for resources. A data URI can be used anywhere that a normal URL would be placed and is functional on client-side as well as server-side. Data URI's can greatly increase the performance of a site.

Understanding user agent detection

A user agent string is the signature of the use being broadcast across the network, information such as browser type, operating syste, platform, and their respective versions and identifying codes. While client-side feature dection is preferable, at times server-side detection may be unavoidable. Limiting server-side detection or using in conjunction with server-side detection is recommended if server-side detection is implemeneted. Using detection capabilities you will be able to ensure the user receieves content that is specific to their features and platform. Libraries like Modernizr once again make this task more streamline. For server-side libraries, use WURFL or Mobi.

HTML5 Mobile Boilerplate is a collection of best practices which acts as a starting point for developers to build on top of. It provides a collection of client-side and server-side code which help pages work on modern mobile devices.

Understanding Mobile Browser Capabilities

Measuring performance

Using analytical tools to measure the performance of the browser when the user interacts with it can provide valuable feedback on site design and functionality. Previously only available with add-ons, browsers are now beginning to recognize supported code which can be used to track site load times, load to user interaction times, mouse tracking, etc. Information is collected in the background and can be collected using scripts and sent back to the web server.

Creating full-screen web apps

In the head of the DOM using the meta and elements along with properly styled CSS we can have our web sites be treated as applications by mobile devices. This includes home screen icons, splash pages, and the removal of an address bar from the browser. It gives web pages a very polished feel and better overall user experience.

Customizing the user interface

Meta tags in the head section also can be used to alter the way the user interacts with the web sites innate functionality. Some attributes include preventing the ability to select elements of the browser, change the highlight color when selecting content, preventing the phone from pulling up the telephone dialer when phone numbers are touched, and also preventing the press-and-hold menu pop-up from occurring.

Detecting orientation changes

JavaScript may be used to code orientation detection which allows sites to recognize when a mobile browser is in portrait mode or landscape mode and adjust its orientation accordingly.

Detecting device movement

JavaScript may also be used to access a mobile devices built in Gryoscope and Accelorometer to detect the pitch, yaw, and roll of the mobile phone. Interval parameters are set during this time to determine if the user to purposely access a built-in application feature or had just happened to drop or move their phone.

Using touch events

The touch interface is a powerful tool which expands on the base functionality of a mouse, being able to touch events rather than click events has led to the development of detecting multiple touch occurrences, swiping motions, pinch, and also pull. This development has expanded over a number of years with the integration of a touchpad on a lap top to the now large smart phone and tablet displays available on the market today.

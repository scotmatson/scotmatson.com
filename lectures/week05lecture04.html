Emphasizing text

To emphasize text four tags exist which seemingly appear simple and perform the same function, they each have additional underlying meaning beyond their visible presentation. The bold tag <b> is a styling tag used to tell the browser the text needs to be displayed in bold but gives no additional meaning to the text. In contrast, the strong tag <strong> also applies a bold appearance to the text but gives it additional meaning that a computer or screen reader would understand. This relationship also exists between the italic tag <i> which is presentational and the <em> which is meaningful and emphasizes the text it wraps.

Displaying special characters

Occasionally the need arises to use reserved symbols that HTML recognizes as parts of it's own language that will not display properly. An example would be when I type in HTML code that is meant to be read as HTML within the browser rather than by the browser. So to make angled brackets appear I need to type &lt; to create a left angled bracket and &rt to create a right angled bracket. If I had not typed these special codes to display these characters HTML would assume I am trying to creating a tag rather than trying to display a tag.

Controlling whitespace

Whitespace is all the empty space that exists between lines of code within tags. In most programming languages whitespace is ignored as whitespace tends to be used to format code to be read easier rather than exist with intent to be presented by the browser. If you would like whitespace to appear in a browser a few options exist. The <pre> tells the browser to display text between the opening and closing brackets exactly as it was written. The other option is a non-breaking space character. &nbsp; will create an additional space within a line of text, although the non-breaking space is not actually designed just to create a space, rather its intention is to be placed between words that you do not want to be separated by the word wrap when the size of the browser window changes. If whitespace needs to be created for stylish reasons it should be done by using CSS, not HTML.

Inserting images

The <img> tag is a self closing tag which is used to place an image upon a page. When the image tag is used it requires the use of multiple attributes to properly upload, size, and give alternate text in case the image does not load. First, src="" is used to give the browser a path to the image so it can upload the image file to the browser. Second the width="" and height="" attributes are used to ensure the image is sized and rendered in the browser properly, if width and height are not used then the native settings of the image will be used. Finally alt="" is used to give a description of the image to the browser which is displayed when the image does not load and also read by screen readers.

An interesting thing about image tags is that the image content displayed as an in-line item rather than a block level element. This means the browser does not distinguish any differences between images or text which appears on a page. While this can cause some unattractive pages at first, it also allows CSS styling to be used easily to word wrap images and manipulate the image location within the browser. In some cases HTML is still used with the align="" attribute to position images but this is only in cases such as HTML eMail, blogs, or forums where CSS styling is not an option. For this reason knowing legacy code is a good idea.

Creating Links

Linking to pages within your site

Covers to the use of the anchor tag to link to other .html files. The anchor tag is created with the element <a> and needs the attributes href="enter link here". Href declares the directory path and stands for hypertext reference, the link in the double quotation can be a absolute path, root-relative path, or a document-relative path. Other attributes one could include would be title="" which creates a pop-up text box on the browsers display when a user hovers the cursor over the link and assists screen readers, also, target="" tells the browser where the new page should open

Linking to external pages

Absolute links are links which redirect a user to .html files which fall outside of the site directory. They use the same anchor tag and href attribute mentioned above, but the path used is what makes this absolute. An absolute path begins with http:// to indicate to the browser that this is an absolute path. The data which follows should include information such as a subdomain, domain, top-level domain, and directories if applicable.

Linking to downloadable resources

To create a path which allows users to download information, as before, you create the anchor tag, href attribute and create a path to the specific file that should be downloaded by the user. These files will be downloaded because they are not .html files and would not be rendered by the browser. So file types such as .zip, .docx, .jpeg, .mp3 will be downloaded when pathed with an anchor tag rather than displayed on the browser itself.

Linking to page regions

In addition to linking to external sites and downloading files, the anchor/href element can be used to navigate within one or multiple pages on a site. This is useful for pages which contain an extensive amount of documentation and require the user to scroll extensively, or for pages which reference to other areas on a site. To accomplish this, id="" attributes should be used to uniquely identify varies points of interest within the document. These id's can later be referenced using the anchor/href element by pathing the id using an octothorpe followed by the id name. These identifiers are referred to as 'fragment identifiers.'

Building Tables

Examining basic table structure

Tables allow us to present data in a non-linear fashion. The basic tags are <table> which is used to create a table, <tr> which defines a table row, <th> defines a table header, and <td> which defines table data, more commonly known as a table cell.

Rows are implicitly created through the use of the table row element, while columns are created by the content that is placed within the table data or table header tag. The browser will automatically create columns as necessary to meet the specifications required by the user.

Adding content to tables

Tables essentially can hold any type of data defined by the user and because of this they are highly versitile elements to include on a web page. Because of this, the tables also will 'flex' to hold content and all of it's predefined elements. This can lead to problems in formatting if care is not taken to remain attentive to the additional padding and margins which certain tags add, or first sizing images to user specificiations prior to use.

Setting table attributes

Certain attributes exist for formatting tables, but most of them have depricated in HTML5. On the table tag these include border="creates a border around the table and each cell.", cellspacing="the space between the table cells.", cellpadding="controls the amount of space within the table cell.", width="sets the width of the table.". On the table data there are colspan="takes up a user defined # of cells to present data for single cell.", and rowspan="takes up a user defined # of rows to present data for a single cell.". These formatting options are now commonly performed using CSS.

Adding table captions

To add captions to a table add the table caption element directly after the table tag <caption>. This should be a very short descriptive text which defines the table. This tag only works for tables.

Defining table headers

The table header tag is used to 'head' each column of a table and define the content which falls below it. The default rendering of this element is center alignment and bold text.
